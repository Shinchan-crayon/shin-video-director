# Shin-video 新手小白安装与制作教程

这份教程给第一次使用 Shin-video 的用户看。你不需要懂 WhisperX、Remotion、Python 或视频渲染，只要按步骤准备环境和材料。

## 一、Shin-video 是怎么做视频的

Shin-video 不是剪辑软件，也不是单纯生成 PPT。它的工作方式是：

```text
文章
→ 口播文案
→ MiniMax 生成口播音频
→ WhisperX 生成音频时间戳
→ Agent 生成视频场景
→ Remotion 视频导演生成镜头方案
→ 选择视频风格和比例
→ Remotion 渲染预览
→ 确认后导出 MP4
```

你只需要参与三件事：

```text
给文章
确认口播文案和音频
确认预览视频
```

## 二、第一次使用前要准备什么

### 1. 准备项目目录

把 Shin-video 项目放到本地，例如：

```bash
cd "/Users/shinchan/Documents/Shin-video"
```

如果你用的是别的目录，把命令里的路径换成你的项目路径。

### 2. 安装基础环境

先检查：

```bash
node -v
npm -v
python3 --version
ffmpeg -version
```

如果缺 Node.js 或 FFmpeg，macOS 推荐：

```bash
brew install node ffmpeg
```

### 3. 安装 Shin-video 前端和 Remotion 依赖

在 Shin-video 项目根目录执行：

```bash
npm install
npx remotion --version
```

能看到 Remotion 版本，说明视频渲染环境基本可用。

### 4. 配置 MiniMax 音频模型

Shin-video 默认使用 MiniMax：

```text
模型：speech-2.8-hd
默认音色：Chinese (Mandarin)_Reliable_Executive
用途：把口播文案生成 口播音频.mp3
```

复制环境变量模板：

```bash
cp .env.local.example .env.local
```

然后打开 `.env.local`，填写自己的 MiniMax API Key：

```text
MINIMAX_API_KEY=你的_minimax_api_key
MINIMAX_TTS_ENDPOINT=https://api.minimax.chat/v1/t2a_v2
MINIMAX_TTS_MODEL=speech-2.8-hd
MINIMAX_VOICE_ID="Chinese (Mandarin)_Reliable_Executive"
```

注意：

```text
不要把真实 API Key 发给别人
不要把 .env.local 上传到 GitHub
如果 MiniMax 控制台要求 GroupId，再填写 MINIMAX_GROUP_ID
```

### 5. 部署 WhisperX

WhisperX 用来把音频变成精准时间戳。第一次使用时，让 Agent 按下面这份文档部署：

```text
references/WhisperX和Remotion部署说明.md
```

部署完成后，`.env.local` 里至少要有：

```text
WHISPERX_PYTHON=/你的/whisperx-env/bin/python
WHISPERX_MODEL=/你的/faster-whisper-medium
```

## 三、启动 Shin-video

在项目根目录执行：

```bash
npm run dev -- --hostname 127.0.0.1 --port 3000
```

浏览器打开：

```text
http://127.0.0.1:3000
```

如果想让同一局域网的其他设备访问，把 `127.0.0.1` 换成 `0.0.0.0`：

```bash
npm run dev -- --hostname 0.0.0.0 --port 3000
```

然后用本机局域网 IP 访问。

## 四、第一次制作视频

### 第一步：开始制作

打开页面后，点击：

```text
开始制作视频
```

### 第二步：输入文章

你可以直接粘贴文章正文，也可以给文章链接。

对 Agent 说：

```text
用 Shin-video 把这篇文章做成一条口播视频。
```

### 第三步：确认口播文案

Agent 会生成 `口播文案.md`。

你要检查：

```text
是否顺口
是否准确
是否有不想要的内容
是否适合短视频朗读
```

确认后说：

```text
确认，继续生成音频。
```

### 第四步：生成并试听音频

Agent 会用 MiniMax 生成：

```text
口播音频.mp3
```

你需要试听。如果满意，说：

```text
确认，继续生成时间戳。
```

如果不满意，可以让 Agent 调整文案、语速、音色或重新生成。

### 第五步：生成时间戳和视频脚本

Agent 会自动执行：

```text
WhisperX 时间戳
字幕节奏整理
视频场景生成
Remotion 视频导演
```

你不需要手动改 JSON 文件。

### 第六步：选择视频风格和比例

Agent 会询问视频风格，默认推荐：

```text
浅色财经 MG
```

比例目前优先支持：

```text
16:9
9:16
```

如果你不知道怎么选，可以直接说：

```text
默认风格，16:9。
```

### 第七步：生成预览视频

Agent 会生成：

```text
runtime/preview.mp4
```

你需要看一遍，重点检查：

```text
字幕是否完整
字幕是否跟音频大致对齐
画面有没有遮挡字幕
画面是不是只在复读字幕
是否黑屏或无声音
```

### 第八步：导出成片

预览满意后说：

```text
确认，导出成片。
```

最终视频会输出到：

```text
成品/成品.mp4
```

## 五、常见问题

### 没有生成音频

检查 `.env.local` 是否配置了：

```text
MINIMAX_API_KEY
MINIMAX_TTS_MODEL
MINIMAX_VOICE_ID
```

### WhisperX 跑不起来

让 Agent 读取：

```text
references/WhisperX和Remotion部署说明.md
```

并先做环境自检，不要跳过时间戳步骤。

### 视频像 PPT 或画面重复

让 Agent 检查：

```text
runtime/director-plan.json
runtime/style-profile.json
remotion 模板是否读取这两个文件
```

没有 `runtime/director-plan.json` 时，不应该直接渲染。

### 字幕和画面重叠

要求 Agent 回到：

```text
Remotion 视频导演
Remotion 渲染成片
```

重新检查字幕安全区和主画面布局。
