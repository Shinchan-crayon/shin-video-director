# Shin-video 前期准备说明

这份说明用于开始使用 Shin-video 工作流之前，确认机器、依赖、素材和部署状态是否准备好。

## 一、这套工作流基于什么制作视频

Shin-video v1 是一套“文章口播驱动的本地 MG 动画视频工作流”。

它基于下面几类能力组合完成视频：

```text
文章内容
→ 文本模型整理和改写为口播文案
→ 用户用音频模型生成口播音频
→ WhisperX 识别音频并生成精准时间戳
→ Agent 根据口播和时间戳生成视频场景脚本
→ Agent 根据用户选择生成视觉风格和动画配置
→ Remotion 把脚本、动画和音频渲染为 MP4
```

核心分工：

| 模块 | 作用 |
|---|---|
| 文本模型 | 整理文章、生成自然口播文案 |
| 音频模型 | 把用户确认后的口播文案生成 `口播音频.mp3` |
| WhisperX | 读取口播音频，生成带时间戳的 `runtime/asr.json` |
| Agent | 调度 Skill，生成 `scenes.json` 和 `style-profile.json` |
| Remotion | 按时间轴、动画配置和音频渲染视频 |
| FFmpeg / 编码环境 | 支撑 MP4 编码和基础音视频检查 |

v1 默认是纯自然语言 MG 动画视频，不把图片链路作为必要条件。

## 二、用户需要准备什么

用户只需要准备：

```text
1. 一篇文章，或者一个文章链接
2. 用户确认后的口播文案
3. 根据口播文案生成的口播音频
```

音频文件统一命名：

```text
口播音频.mp3
```

用户不需要学习：

```text
WhisperX
Remotion
Python
Node.js
FFmpeg
scenes.json
style-profile.json
渲染命令
```

这些由 Agent 按工作流处理。

## 三、机器需要什么环境

必须具备：

```text
macOS 或可运行 Node.js / Python / FFmpeg 的本地机器
Node.js
npm
Python 3.10 或 3.11
FFmpeg
WhisperX
faster-whisper 模型

```

推荐检查命令：

```bash
node -v

```

## 四、需要安装什么

### 1. Node.js 和 npm

用于运行 Shin-video 项目和 Remotion。

macOS 推荐：

```bash
brew install node
```

验证：

```bash
node -v
npm -v
```

### 2. FFmpeg

用于音视频编码和检查。

macOS 推荐：

```bash
brew install ffmpeg
```

验证：

```bash
ffmpeg -version
```

### 3. Remotion

用于把 React 视频模板渲染成 MP4。

如果已经有 Shin-video 项目，进入项目根目录：

```bash
npm install
npx remotion --version
```

如果缺 Remotion：

```bash
npm install remotion @remotion/cli @remotion/renderer @remotion/bundler
```

如果要从零新建 Remotion 项目，可使用官方脚手架：

```bash
npx create-video@latest
```

Shin-video 正式运行时，仍然使用 Shin-video 自己的视频模板。

### 4. Python 和 WhisperX

用于读取 `口播音频.mp3` 并生成精准时间戳。

推荐独立环境：

```bash
mkdir -p "$HOME/Documents/Ai剪辑/asr_runtime"
cd "$HOME/Documents/Ai剪辑/asr_runtime"
python3 -m venv whisperx-env
source whisperx-env/bin/activate
python -m pip install --upgrade pip setuptools wheel
pip install whisperx
```

验证：

```bash
python -m whisperx --help
```

### 5. faster-whisper 模型

推荐路径：

```text
$HOME/Documents/Ai剪辑/asr_runtime/models/faster-whisper-medium
```

下载示例：

```bash
source "$HOME/Documents/Ai剪辑/asr_runtime/whisperx-env/bin/activate"
pip install -U huggingface_hub
mkdir -p "$HOME/Documents/Ai剪辑/asr_runtime/models"
huggingface-cli download Systran/faster-whisper-medium \
  --local-dir "$HOME/Documents/Ai剪辑/asr_runtime/models/faster-whisper-medium"
```

如果用户已有模型，优先使用已有模型路径。

## 五、需要部署什么

Agent 需要部署或确认三部分：

### 1. Shin-video 工作流包

包括：

```text
workflow-manifest.json
AGENT-先读我.md
skills/
references/
templates/
```

作用：

```text
让 Agent 知道整个工作流如何调度，每个 Skill 负责什么。
```

### 2. WhisperX 本地 ASR 环境

包括：

```text
Python 虚拟环境
whisperx Python 包
faster-whisper 模型
FFmpeg
```

作用：

```text
把 口播音频.mp3 转成 runtime/asr.json。
```

### 3. Remotion 本地渲染环境

包括：

```text
Node.js
npm
Remotion
Shin-video 视频模板
FFmpeg / 编码环境
```

作用：

```text
把 runtime/scenes.json、runtime/style-profile.json 和 口播音频.mp3 渲染为 MP4。
```

## 六、推荐配置文件

项目根目录可放 `.env.local`：

```bash
WHISPERX_PYTHON="$HOME/Documents/Ai剪辑/asr_runtime/whisperx-env/bin/python"
WHISPERX_MODEL="$HOME/Documents/Ai剪辑/asr_runtime/models/faster-whisper-medium"
WHISPERX_LANGUAGE="zh"
REMOTION_ENTRY="remotion/index.ts"
REMOTION_COMPOSITION_ID="ShinVideo"
```

读取优先级：

```text
1. 项目 .env.local
2. 用户本机已有路径
3. Shin-video 默认路径
4. 找不到就提示用户缺什么，不要假装已部署
```

## 七、开始前自检清单

开始跑完整工作流前，Agent 必须确认：

- [ ] 能读取文章或文章链接
- [ ] 能生成并保存 `文章正文.md`
- [ ] 能生成并保存纯净 `口播文案.md`
- [ ] 用户已放入 `口播音频.mp3`
- [ ] `node -v` 正常
- [ ] `npm -v` 正常
- [ ] `python3 --version` 正常
- [ ] `ffmpeg -version` 正常
- [ ] `python -m whisperx --help` 正常
- [ ] faster-whisper 模型路径存在
- [ ] `npx remotion --version` 正常
- [ ] Shin-video Remotion 模板存在

## 八、输出文件约定

项目目录保持：

```text
视频项目/
├── 文章正文.md
├── 口播文案.md
├── 口播音频.mp3
├── runtime/
│   ├── asr.json
│   ├── scenes.json
│   ├── style-profile.json
│   └── preview.mp4
└── 成品/
    └── 成品.mp4
```

硬性规则：

```text
中间文件放 runtime/
预览视频放 runtime/preview.mp4
最终成片放 成品/成品.mp4
不要把临时文件散落在项目根目录
```

## 九、不能跳过的关键点

- 不能跳过 WhisperX 时间戳。
- 不能手工编时间戳冒充 ASR。
- 不能把分镜、画面提示写进 `口播文案.md`。
- 不能让用户必须理解内部 JSON 文件。
- 不能把最终视频输出到 `成品/` 之外。

## 十、工作流一句话说明

```text
Shin-video 是基于文本模型、音频模型、WhisperX 精准时间戳和 Remotion React 视频渲染搭建的本地口播视频生成工作流，用来把一篇文章自动生成浅色 MG 动画短视频。
```
